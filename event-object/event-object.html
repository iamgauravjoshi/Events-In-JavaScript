üìãEvent Object Properties

The Event Object has many properties that give you detailed information about what happened:

type string

The name of the event (e.g., "click", "keydown", "mouseover")

event.type // "click"

target Element

The element that triggered the event

event.target // <button>...</button>

currentTarget Element

The element the event listener is attached to

event.currentTarget // listener element

timeStamp number

When the event occurred (milliseconds since page load)

event.timeStamp // 1234.567

bubbles boolean

Whether the event bubbles up through the DOM

event.bubbles // true or false

cancelable boolean

Whether the event can be cancelled

event.cancelable // true or false

üñ±Ô∏èMouse Event Properties

Mouse events have additional properties specific to mouse interactions:

clientX, clientY number

Mouse position relative to the browser window

event.clientX // 150
event.clientY // 200

pageX, pageY number

Mouse position relative to the entire page

event.pageX // 150
event.pageY // 800

screenX, screenY number

Mouse position relative to the user's screen

event.screenX // 500
event.screenY // 300

button number

Which mouse button was pressed (0=left, 1=middle, 2=right)

event.button // 0, 1, or 2

buttons number

Bitmask of which buttons are currently pressed

event.buttons // 1, 2, 4, etc.

altKey, ctrlKey, shiftKey boolean

Whether modifier keys were pressed

event.ctrlKey // true/false

‚å®Ô∏èKeyboard Event Properties

Keyboard events provide detailed information about key presses:

key string

The actual key value (what you see)

event.key // "a", "Enter", "ArrowUp"

code string

The physical key code (keyboard layout independent)

event.code // "KeyA", "Enter", "ArrowUp"

keyCode number

Numeric key code (deprecated but still used)

event.keyCode // 65, 13, 38

repeat boolean

Whether the key is being held down

event.repeat // true/false
