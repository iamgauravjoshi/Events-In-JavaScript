🤔What are JavaScript Events?

Imagine a webpage as a playground. Events are the actions that happen in this playground and web pages can "listen" for things happening and react to them.

Without events, a web page is just static. Events make it interactive — just like you pressing buttons in a lift, and the lift responding!

Simple Example:
When you click a button, that's a "click event".
When you type in a text box, that's a "keypress event".
When you move your mouse, that's a "mousemove event".

📝Common Types of JavaScript Events

There are many types of events, but here are the most common ones you'll use:

🖱️ Mouse Events

  • click - When you click something
  • dblclick - double click event
  • mouseover - When you hover over something
  • mouseout - When you move mouse away
  • mousemove - When you move the mouse

⌨️ Keyboard Events

  • keydown - When you press a key
  • keyup - When you release a key
  • keypress - When you type a character

📋 Form Events

  • submit - When a form is submitted
  • change - When input value changes
  • focus - When you click in an input
  • blur - When you click away from input

🌐 Window Events

  • load - When page finishes loading
  • resize - When window size changes
  • scroll - When you scroll the page

👂 JavaScript Event Handling Methods

To make your webpage respond to events, you need to set up "event listeners". Think of them as guards who watch for specific things to happen.

Three ways to add event listeners:
1. HTML way: onclick="doSomething()" (like we've been using)
2. JavaScript property: button.onclick = function() { ... }
3. addEventListener: button.addEventListener('click', function() { ... })
